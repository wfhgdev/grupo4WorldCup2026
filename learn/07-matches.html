⚽ Página de Partidos

📖 Para principiantes:
  • Skeleton loader — placeholder mientras los datos cargan. Bloques grises animados que se parecen al contenido.
  • Filtración — mostrar solo los elementos necesarios. En el proyecto: filtro por estado (live, finished, scheduled).
  • querySelectorAll — buscar elementos por selector CSS. Devuelve un NodeList (parecido a un array).
  • createElement — crear un nuevo elemento HTML en JS.
  • appendChild — agregar el elemento creado al DOM.

¿Qué hace esta página?

Muestra los partidos filtrados por estado y con un estado visual claro. Fuentes: [Fuente: ../matches/matches.html] y [Fuente: ../matches/matches.js].

Skeleton Loaders

Los skeletons evitan pantalla en blanco mientras cargan datos. Ejemplo:

function renderSkeletons(container) {
  for (let i = 0; i < 6; i++) {
    const skeleton = document.createElement('article');
    skeleton.className = 'bg-surface-container-lowest ...';
    skeleton.innerHTML = '<div class="skeleton-shimmer">...</div>';
    container.appendChild(skeleton);
  }
}

🧱 Mental model: Skeleton = maqueta vacía que parece contenido, para que el usuario no piense que se rompió la página.

Tarjetas de Partido

Dependiendo del estado, el foco visual cambia: marcador y minuto en vivo, resultado final en finalizados, hora en programados.

  • Finalizado — Resultado y posible penaltis.
  • Programado — Hora y fecha relativa.

⚽ Matches Page

📖 For beginners:

  • Filtering — show only needed elements. In the project: filter by match status (live, finished, scheduled).
  • querySelectorAll — find all elements by CSS selector. Returns a NodeList (array-like).
  • createElement — create a new HTML element in JS.
  • appendChild — add the created element to the DOM.

What it does
