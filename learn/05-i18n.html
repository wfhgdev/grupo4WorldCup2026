📖 Para principiantes: conceptos básicos

  1. i18n — abreviatura de "internacionalización" (18 letras entre i y n). Sistema de traducción sin duplicar código.
  2. Diccionario (dictionary) — objeto donde clave = código, valor = traducción.
    { "BRA": "Brasil", "ARG": "Argentina" }
  3. NAME_TO_TLA — diccionario inverso: nombre completo → código FIFA.
    { "Brazil": "BRA", "Argentina": "ARG" }
  4. Función de traducción t(key) — busca una clave en el diccionario. Si no la encuentra, devuelve la misma clave.
  5. 4 archivos de diccionario: teams.js (equipos), enums.js (estados), competition.js (nombres de torneos), ui.js (textos de interfaz).
💡 Consejo: empieza con teams.js y enums.js — cubren el 80% de las traducciones visibles.

Cómo funciona

Un diccionario asocia claves a traducciones:

window.__DICT = {
  BRA: 'Brasil',
  ARG: 'Argentina',
  ESP: 'España'
};

function t(key) {
  return dict[key] || key;
}

t('BRA'); // "Brasil"

🌐 Internationalization (i18n)

📖 For beginners: basic concepts

  1. i18n — abbreviation for "internationalization" (18 letters between i and n). A translation system without duplicating code.
  2. Dictionary — an object where key = code, value = translation.
    { "BRA": "Brazil", "ARG": "Argentina" }
  3. NAME_TO_TLA — reverse dictionary: full name → FIFA code.
    { "Brazil": "BRA", "Argentina": "ARG" }
  4. Translation function t(key) — looks up a key in the dictionary. If not found, returns the key itself.
  5. 4 dictionary files: teams.js (teams), enums.js (statuses), competition.js (tournament names), ui.js (interface texts).
💡 Tip: start with teams.js and enums.js — they cover 80% of visible translations.
