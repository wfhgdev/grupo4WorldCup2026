🛠️ Cómo Reconstruir Este Proyecto

📖 Para principiantes:
  • Servidor local — necesitas servir los archivos por HTTP (no file://) porque el footer y el proxy usan rutas absolutas (/components/..., /api/proxy). npx serve . o python -m http.server 8000 alcanzan para desarrollar.
  • Serverless (Vercel) — funciones que se ejecutan bajo demanda en la nube, sin mantener un servidor propio. El proxy real del proyecto (api/proxy.js) es una función serverless de Vercel.
  • proxy.php — versión legada del proxy en PHP. Se conserva en la raíz, pero el cliente actual no la usa; solo sirve como referencia si algún día despliegas en hosting PHP.

Guía paso a paso para crear tu propia versión desde cero. Cada paso referencia archivos reales del proyecto.

Paso 7: Proxy (Serverless)

Crea api/proxy.js como función serverless (para Vercel) — es la que consume el cliente. Fuente: [Fuente: ../api/proxy.js].

export default async function handler(req, res) {
  const endpoint = req.query.endpoint || '';
  const token = process.env.API_TOKEN; // variable de entorno en Vercel
  const url = 'https://api.football-data.org/v4' + endpoint;

  const response = await fetch(url, {
    headers: { 'X-Auth-Token': token }
  });
  const data = await response.json();
  res.status(response.status).json(data);
}
📁 ¿Y proxy.php? Es una versión legada del mismo proxy escrita en PHP, guardada en la raíz del proyecto. js/api.js nunca la llama — solo existe como alternativa si en algún momento despliegas en un hosting que solo soporte PHP (por ejemplo, sin funciones serverless).

Paso 12: Probar localmente

npx serve .

Abre http://localhost:3000. También sirve python -m http.server 8000 si tienes Python. Cualquiera de los dos es obligatorio para que el footer (fetch a /components/...) funcione — con file:// no cargará.

Paso 13: Deploy

Vercel es la opción recomendada: despliega api/proxy.js como función serverless automáticamente, sin configuración extra, y ahí es donde defines la variable de entorno con el token de football-data.org.

GitHub Pages es solo hosting estático — no ejecuta funciones serverless ni PHP, así que api/proxy.js (y proxy.php) simplemente no funcionarían ahí. Si necesitas GitHub Pages, tendrías que sustituir el proxy propio por un servicio externo (por ejemplo corsproxy.io) o aceptar que la sección de datos en vivo no funcione.

🛠️ How to Rebuild This Project

📖 For beginners:
  • Local server — you must serve files over HTTP (not file://) because the footer and proxy use absolute paths (/components/..., /api/proxy). npx serve . or python -m http.server 8000 are enough for development.
  • Serverless (Vercel) — on-demand cloud functions with no server to maintain. The project's real proxy (api/proxy.js) is a Vercel serverless function.
  • proxy.php — a legacy PHP version of the proxy. It's kept at the project root, but the current client never calls it; it only exists as a fallback for PHP-only hosting.

Use these steps and compare with the real project sources.

Step 1: Structure

Create folders: api/, js/i18n/es, components/footer/, matches, groups, knockout, stats. There is no home.js — the home page is static.

Step 2: Base HTML

Use [Source: ../index.html] as structural reference: header, main, and a <div id="footerInjection"> instead of a hardcoded footer.

Step 3: Tailwind

Load the Tailwind CDN + [Source: ../js/tailwindCustom.js] (note: it lives under js/, not at the root).

Tailwind — Utility-First, Tailwind — CDN

Step 4: Mobile menu

See js/menu.js for the IIFE + scroll lock. Load it on every page, including home.

Step 5: Shared footer & cookie consent

Build components/footer/componentFooter.html and inject it with footerInjection.js, then add cookieConsent.js. Full details in Lesson 12. Requires an HTTP server — file:// breaks the fetch.

Step 6: Team crests

Build js/teamImage.js plus the .crest-* CSS classes in index.css. See Lesson 12.

Step 7: Proxy (Serverless)

See api/proxy.js — the one the client actually calls. Get a token at football-data.org. proxy.php is legacy and unused by the client.

Step 8: API + cache + i18n

js/api.js and js/i18n/i18n.js — remember i18n here translates API data, it's not a UI language switcher.

Step 9: Deploy

Prefer Vercel: it deploys api/proxy.js as a serverless function automatically and is where you set the API token as an environment variable.

GitHub Pages is static-only — it cannot run serverless functions or PHP, so the proxy simply won't work there without an external CORS proxy alternative.
