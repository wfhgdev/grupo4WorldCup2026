🎨 HTML, CSS & Tailwind

📖 Para principiantes: conceptos básicos

  1. ¿Qué es una etiqueta HTML? — Es una instrucción para el navegador: <p> significa "aquí empieza un párrafo", </p> significa "el párrafo terminó". La mayoría de etiquetas vienen en pares: apertura + cierre.
    Ejemplos: <h1>Título</h1> — encabezado grande. <p>Texto</p> — párrafo. <a href="...">Enlace</a> — enlace.
  2. ¿Qué es un atributo? — Información adicional dentro de una etiqueta: href="url", class="nombre", id="identificador". Se escriben dentro de la etiqueta de apertura.
  3. ¿Qué es un selector CSS? — Una regla que dice "aplica este estilo a estos elementos".
    .clase { color: red; } — todos los elementos con class="clase"
    #id { color: red; } — el elemento con id="id"
    h1 { color: red; } — todas las etiquetas <h1>
  4. ¿Qué es una clase de Tailwind? — CSS predefinido empaquetado en un nombre corto.
    text-center = text-align: center
    bg-blue-500 = background-color: #3b82f6
    p-4 = padding: 1rem
    rounded-lg = border-radius: 0.5rem
    flex = display: flex
  5. CDN — Content Delivery Network. Biblioteca cargada por internet en lugar de almacenarla en tu propio servidor. Ahorra espacio y acelera la carga.
  6. Responsive — Adaptación al tamaño de pantalla. mobile-first: primero estilos para móvil, luego para tablet (md:), luego para ordenador (lg:).

HTML: el esqueleto de la página

HTML define qué hay en la página: encabezados, textos, botones, imágenes. En el proyecto este esqueleto lo revisamos en [Fuente: ../index.html]. Cada elemento empieza con una etiqueta, puede cerrarse, y los atributos describen su comportamiento o apariencia básica.

<!doctype html>
<html lang="es">
  <head>
    <meta charset="utf-8"/>
    <title>FIFA WC 2026</title>
    <link rel="stylesheet" href="index.css"/>
  </head>
  <body>
    <header>...</header>
    <main>...</main>
    <footer>...</footer>
  </body>
</html>

Para principiantes: empieza por head y body. Dentro de body, además, trata de usar etiquetas semánticas como header, main, section, footer porque mejoran accesibilidad y SEO.

Recurso: MDN — HTML.

⚠️ Error común: olvidar cerrar etiquetas o poner &lt;/div&gt; en el lugar equivocado. Revisa la indentación y coincide aperturas y cierres.

CSS: cómo se ve cada elemento

CSS controla colores, tipografía, espaciado y disposición. En el proyecto los estilos personalizados están en [Fuente: ../index.css]. Una regla se compone de selector + declaraciones.

/* selector + declaraciones */
.gradient-btn {
  background: linear-gradient(45deg, #4f378a, #6750a4);
  transition: all 0.3s ease;
}

/* estado hover */
.gradient-btn:hover {
  filter: brightness(1.1);
  transform: translateY(-2px);
}

Regla práctica: empieza con clases reutilizables y usa combinadores simples. No crees reglas muy largas: si se complica, probablemente es mejor moverla a una clase específica o usar utilidades.

Recurso: MDN — CSS.

💡 Consejo: sigue el orden base → responsive → estado (hover/focus). Así evitas peleas entre reglas.

Tailwind: utilidades dentro del HTML

Tailwind permite estilizar rápido con clases pequeñas en vez de inventar nombres nuevos en cada archivo. La configuración del proyecto está en [Fuente: ../tailwindCustom.js], donde definimos paleta, fuentes y espaciado.

<button class="bg-[#4f378a] text-white p-4 rounded-lg">
  Haz clic
</button>

En este proyecto comparten la misma paleta Material-like en varias páginas, por eso se siente coherente el diseño.

Recursos: Tailwind — Utility-First, Tailwind — CDN.

💡 Consejo: en Tailwind la última clase gana cuando hay conflicto. Ejemplo: text-red-500 text-blue-600 → quedará azul.

Responsive: mobile first

Los estilos base aplican en móvil y luego sobreescribes en pantallas más grandes:

  • lg: ≥ 1024px
  • md: ≥ 768px
  • Sin prefijo = base, primero móvil

Ejemplo real desde [Fuente: ../index.html]:

<nav class="hidden lg:flex gap-gutter">
  <!-- Oculto en móvil, visible en desktop -->
</nav>

Recurso: Tailwind — Responsive design.

Dark mode

El proyecto soporta modo oscuro. En Tailwind usamos variantes con prefijo dark: y estilos adicionales según la variante activa.

<div class="bg-surface dark:bg-surface-dim">
  <!-- Claro por defecto, oscuro cuando hay modo oscuro -->
</div>

Recurso: Tailwind — Dark mode.

🎨 HTML, CSS & Tailwind

📖 For beginners: basic concepts

  1. What is an HTML tag? — A command for the browser: <p> means "a paragraph starts here", </p> means "the paragraph ended". Most tags come in pairs: opening + closing.
    Examples: <h1>Heading</h1> — large heading. <p>Text</p> — paragraph. <a href="...">Link</a> — link.
  2. What is an attribute? — Extra information inside a tag: href="url", class="name", id="identifier". Written inside the opening tag.
  3. What is a CSS selector? — A rule that says "apply this style to these elements".
    .class { color: red; } — all elements with class="class"
    #id { color: red; } — the element with id="id"
    h1 { color: red; } — all <h1> tags
  4. What is a Tailwind class? — Prebuilt CSS packed into a short name.
    text-center = text-align: center
    bg-blue-500 = background-color: #3b82f6
    p-4 = padding: 1rem
    rounded-lg = border-radius: 0.5rem
    flex = display: flex
  5. CDN — Content Delivery Network. A library loaded over the internet instead of being stored on your own server. Saves space and speeds up loading.
  6. Responsive — Adaptation to screen size. mobile-first: styles for mobile first, then tablet (md:), then desktop (lg:).

HTML: page skeleton

HTML defines what is on the page: headings, text, buttons, images. Review source in [Source: ../index.html].

<!doctype html>
<html lang="es">
  <head>

    <title>FIFA WC 2026</title>
    <link rel="stylesheet" href="index.css"/>
  </head>
  <body>
    <header>...</header>

For beginners: start with head and body. Inside body, use semantic tags like header, main, section, footer — they improve accessibility and SEO.

⚠️ Common mistake: forgetting to close tags or putting </div> in the wrong place. Check indentation and match opening/closing tags.

CSS: appearance

CSS controls colors, typography, spacing and layout. Custom styles live in [Source: ../index.css]. A rule consists of selector + declarations.

/* selector + declarations */
.gradient-btn {
  background: linear-gradient(45deg, #4f378a, #6750a4);
  transition: all 0.3s ease;
}

/* hover state */
.gradient-btn:hover {
  filter: brightness(1.1);
  transform: translateY(-2px);
}

Start with reusable classes and simple combinators. Don't create very long rules.

MDN — CSS.

💡 Tip: follow the order base → responsive → state (hover/focus). This avoids conflicts between rules.

Tailwind: utilities inside HTML

Tailwind lets you style quickly with small classes instead of inventing new names. Configuration lives in [Source: ../tailwindCustom.js], where we define palette, fonts and spacing.

<button class="bg-[#4f378a] text-white p-4 rounded-lg">
  Click me
</button>

All pages share the same Material-like palette, making the design feel consistent.

Tailwind — Utility-First, Tailwind — CDN.

💡 Tip: in Tailwind, the last class wins when there's a conflict. Example: text-red-500 text-blue-600 → blue wins.

Base styles apply on mobile, then override on larger screens:

  • lg: ≥ 1024px
  • md: ≥ 768px
  • No prefix = base, mobile first

  <!-- Hidden on mobile, visible on desktop -->
