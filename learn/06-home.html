🏠 Página de Inicio

📖 Para principiantes: conceptos clave

  1. Hero section — La primera pantalla del sitio. Debe explicar inmediatamente de qué trata la web.
  2. CSS Grid — Forma de organizar elementos en una cuadrícula.
    grid-template-columns: repeat(12, 1fr) — 12 columnas iguales.
  3. Bento Grid — Cuadrícula de tarjetas de diferentes tamaños, como una caja japonesa bento.
  4. Rutas relativas — Desde learn/ hacia la raíz: ../ → subir un nivel.

Estructura General

La página principal usa una estructura clara pensada para storytelling y navegación. Código real del proyecto en [Fuente: ../index.html]:

  • Header — Navegación superior, en desktop incluye enlaces directos y en móvil el botón hamburguesa que abre el menú overlay.
  • Hero — Imagen grande, gradiente y título principal con llamado a la acción.
  • Bento Grid — Tarjetas hacia Partidos, Grupos, Eliminatorias y Estadísticas.
  • Footer — Información legal y secundaria.
🧱 Para recordar: La página de inicio es estática — no hay ningún home.js ni llamadas a la API. El único contenido dinámico es el footer, inyectado por fetch, y el aviso de cookies.

Scripts cargados en la home

Al final de [Fuente: ../index.html] se cargan tres scripts compartidos, en este orden:

<div id="footerInjection"></div>
<script src="components/footer/footerInjection.js"></script>
<script src="js/cookieConsent.js"></script>
<script>initCookieConsent();</script>
<script src="js/menu.js"></script>
  • footerInjection.js — hace fetch() del footer compartido y lo inyecta en #footerInjection.
  • cookieConsent.js — muestra el banner de cookies si el usuario no ha decidido aún.
  • menu.js — activa el menú móvil (overlay hamburguesa).

Detalle completo de estos tres componentes en la Lección 12 · Componentes Compartidos.

Hero Section

El objetivo del hero es fijar el tema del sitio. La técnica usada es imagen de fondo + capa de gradiente sobre texto.

Ejemplo conceptual desde [Fuente: ../index.html]:

<section class="relative w-full h-[600px] md:h-[700px] flex items-center justify-center overflow-hidden">
  <div class="absolute inset-0 bg-cover bg-center" style="background-image: url('...')"></div>
  <div class="absolute inset-0 bg-gradient-to-t from-black/70 to-transparent"></div>
  <div class="relative z-10 text-center ...">
    <h1>COPA MUNDIAL 2026...</h1>
  </div>
</section>

z-10/z-20 colocan capas. Docs recomendadas: Tailwind — position, Tailwind — background-image, MDN — fondos en CSS.

Bento Grid

El grid usa CSS Grid con columnas adaptativas. Ejemplo desde [Fuente: ../index.html]:

<div class="grid grid-cols-1 md:grid-cols-12 gap-gutter">
  <!-- Partidos -->
  <a href="matches/matches.html" class="col-span-1 md:col-span-6">...</a>
  <!-- Grupos -->
  <a href="groups/groups.html" class="col-span-1 md:col-span-3">...</a>
  <!-- Eliminatorias -->
  <a href="knockout/knockout.html" class="col-span-1 md:col-span-3">...</a>
  <!-- Estadísticas -->
  <a href="stats/stats.html" class="col-span-1 md:col-span-12">...</a>
</div>
💡 Ojo con las rutas: estas rutas (matches/matches.html, sin ../) son las que usa index.html porque vive en la raíz del proyecto. No las confundas con las rutas dentro de learn/, que sí necesitan ../ para subir un nivel (siguiente sección).

Recursos: Tailwind — Grid, MDN — CSS Grid.

💡 Para principiantes: grid-cols-1 = 1 columna en móvil. md:grid-cols-12 = 12 columnas en desktop. col-span-6 = la tarjeta ocupa la mitad.

Rutas desde learn/

Un detalle fácil de olvidar: desde learn/ los enlaces usan ../ para subir a la raíz. Por ejemplo:

  • ../index.html
  • ../matches/matches.html
  • ../js/menu.js

Esto es importante para no crear rutas rotas cuando abras los archivos como file:///C:/.../learn/06-home.html.

Recursos recomendados

  • Tailwind — Grid
  • Web.dev — Guías prácticas

🏠 Home Page

📖 For beginners: key concepts

  1. Hero section — The first screen of a website. It should immediately explain what the site is about.
  2. CSS Grid — A way to arrange elements in a table-like structure.
    grid-template-columns: repeat(12, 1fr) — 12 equal columns.
  3. Bento Grid — A grid of different-sized cards, like a Japanese bento box.
  4. Relative paths — From learn/ to root: ../ → go up one level.

General structure

Header, hero, bento grid and footer. The home page is static — there is no home.js and no API calls. Source: [Source: ../index.html].

Hero

Background image + gradient overlay + headline. Docs: Tailwind — background-image, MDN — backgrounds.

Bento Grid

Links to 4 sections with responsive column spans (root-relative paths like matches/matches.html, since index.html lives at the project root). Docs: Tailwind — Grid, MDN — CSS Grid.

Scripts loaded on the home page

footerInjection.js (fetches the shared footer), cookieConsent.js (cookie banner) and menu.js (mobile menu). Full details in Lesson 12 · Shared Components.

Paths

From learn/, use ../ prefixes so links work locally.
