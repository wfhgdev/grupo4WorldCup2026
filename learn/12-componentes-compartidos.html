🧩 Componentes Compartidos

📖 Para principiantes:
  • Componente — pieza de UI reutilizable que se repite en varias páginas sin copiar y pegar todo el HTML.
  • Inyección por fetch — traer un fragmento HTML con fetch() y meterlo en el DOM con innerHTML.
  • MutationObserver — API del navegador que "observa" cambios en el DOM y ejecuta código cuando ocurren (útil cuando el contenido llega después, de forma asíncrona).
  • onerror (en imágenes) — atributo/evento que se dispara si una imagen no carga; sirve para mostrar un respaldo.

¿Qué hay en esta lección?

Tres piezas pequeñas que se repiten en todas las páginas del sitio (inicio, partidos, grupos, eliminatorias, estadísticas): el footer compartido, el aviso de cookies y los escudos de equipo. Ninguna depende de un framework — son fetch, DOM y CSS puro.

1. Footer inyectado por fetch

En vez de repetir el HTML del footer en cada página, se guarda una sola vez en [Fuente: ../components/footer/componentFooter.html] y se inyecta con un script compartido:

// components/footer/footerInjection.js
fetch("/components/footer/componentFooter.html")
  .then((response) => response.text())
  .then((data) => {
    document.getElementById("footerInjection").innerHTML = data;
  })
  .catch((error) => {
    console.error("Error al cargar el pie de página:", error);
  });

Cada página solo necesita un contenedor vacío y el script:

<div id="footerInjection"></div>
<script src="/components/footer/footerInjection.js"></script>
⚠️ Requiere un servidor HTTP: la ruta empieza por / (raíz del sitio), así que fetch() necesita un origen HTTP real — http://localhost:.... Si abres el archivo directamente con doble clic (file:///C:/.../index.html), el navegador bloquea la petición y el footer no aparece. Usa npx serve . o python -m http.server 8000 desde la raíz del proyecto.

Fuente: [Fuente: ../components/footer/footerInjection.js].

2. Aviso de cookies

js/cookieConsent.js muestra un banner accesible la primera vez que alguien visita el sitio, y recuerda la decisión en localStorage bajo la clave wc2026_cookie_consent:

const STORAGE_KEY = 'wc2026_cookie_consent';

const getConsent = () => localStorage.getItem(STORAGE_KEY);
const saveConsent = (value) => localStorage.setItem(STORAGE_KEY, value); // 'accepted' | 'rejected'

const initCookieConsent = () => {
  if (!getConsent()) showConsentBanner();
  // ...conecta el botón "Cookies" del footer para reabrir el banner
};

El botón Cookies del footer permite reabrir el banner en cualquier momento, incluso después de haber decidido. Como el footer se inyecta de forma asíncrona (ver punto 1), el script usa un MutationObserver para "esperar" a que el botón aparezca en el DOM:

if (document.getElementById('footerInjection')) {
  const observer = new MutationObserver(() => {
    setupFooterCookiesBtn();
  });
  observer.observe(document.getElementById('footerInjection'), { childList: true, subtree: true });
  setTimeout(() => observer.disconnect(), 5000); // se desconecta solo
}
🔒 Es educativo, no seguimiento real: el sitio no usa cookies de publicidad ni analítica de terceros. Solo se guarda la preferencia del usuario y un caché técnico (Lección 04) — el banner existe con fines didácticos de transparencia, no porque haya rastreo que ocultar.

Se activa en cada página con:

<script src="js/cookieConsent.js"></script>
<script>initCookieConsent();</script>

Fuente: [Fuente: ../js/cookieConsent.js].

3. Escudos de equipo (crests)

js/teamImage.js centraliza cómo se pinta el escudo de un equipo en todo el sitio, para no repetir tamaños ni lógica de error en cada página:

const CREST_SIZE_MAP = {
  groups: 'crest-sm',
  matches: 'crest-lg',
  knockout: 'crest-sm',
  knockoutFinal: 'crest-md',
  stats: 'crest-sm'
};

function getTeamCrestHtml(team, context, fallbackText, altText) {
  const sizeClass = CREST_SIZE_MAP[context] || CREST_SIZE_MAP.stats;
  const url = team?.crest || '';
  if (!url) {
    return `<div class="crest-container ${sizeClass}"><span class="crest-fallback">${fallbackText}</span></div>`;
  }
  return `<div class="crest-container ${sizeClass}">
    <img src="${url}" alt="${altText}" class="crest-img"
      onerror="this.onerror=null;this.outerHTML='<span class=\\'crest-fallback\\'>${fallbackText}</span>'">
  </div>`;
}

Cada página pasa un context distinto ('groups', 'matches', 'knockout', 'knockoutFinal', 'stats') para obtener el tamaño correcto sin duplicar CSS. Los tamaños reales están en [Fuente: ../index.css]: .crest-sm (24px), .crest-md (40px), .crest-lg (56px).

Cuando la imagen no existe o falla, onerror la reemplaza por un <span class="crest-fallback"> con la inicial o código del equipo — así nunca se ve un ícono de imagen rota.

💡 Dos variantes de la misma idea: getTeamCrestHtml() devuelve un string HTML (para meter dentro de template literals, como en Partidos y Grupos). createTeamCrest() hace lo mismo pero devuelve un elemento <div> real, útil cuando ya tienes una tarjeta en el DOM y quieres reemplazar un nodo (como en Eliminatorias, con replaceWith()).

Fuente: [Fuente: ../js/teamImage.js].

🧩 Shared Components

📖 For beginners:
  • Component — a reusable piece of UI shared across pages without copy-pasting the whole HTML.
  • Fetch injection — fetching an HTML fragment and inserting it into the DOM via innerHTML.
  • MutationObserver — browser API that watches for DOM changes and runs code when they happen (useful when content arrives asynchronously).
  • onerror (on images) — fires when an image fails to load; used to show a fallback.

What's in this lesson?

Three small pieces shared by every page (home, matches, groups, knockout, stats): the shared footer, the cookie banner, and team crests. None of them depend on a framework — just fetch, DOM and plain CSS.

1. Fetch-injected footer

Instead of repeating the footer markup on every page, it's stored once in [Source: ../components/footer/componentFooter.html] and injected by a shared script:

// components/footer/footerInjection.js
fetch("/components/footer/componentFooter.html")
  .then((response) => response.text())
  .then((data) => {
    document.getElementById("footerInjection").innerHTML = data;
  })
  .catch((error) => {
    console.error("Error loading footer:", error);
  });

Each page just needs an empty container plus the script:

<div id="footerInjection"></div>
<script src="/components/footer/footerInjection.js"></script>
⚠️ Requires an HTTP server: the path starts with / (site root), so fetch() needs a real HTTP origin — http://localhost:.... Double-clicking the file (file:///C:/.../index.html) makes the browser block the request and the footer never appears. Use npx serve . or python -m http.server 8000 from the project root.

Source: [Source: ../components/footer/footerInjection.js].

2. Cookie consent

js/cookieConsent.js shows an accessible banner on first visit and remembers the choice in localStorage under wc2026_cookie_consent. The footer's Cookies button re-opens the banner at any time. Since the footer loads asynchronously (see point 1), the script uses a MutationObserver to wait for that button to appear:

if (document.getElementById('footerInjection')) {
  const observer = new MutationObserver(() => {
    setupFooterCookiesBtn();
  });
  observer.observe(document.getElementById('footerInjection'), { childList: true, subtree: true });
  setTimeout(() => observer.disconnect(), 5000); // auto-disconnects
}
🔒 Educational, not real tracking: the site uses no advertising or third-party analytics cookies. Only the user's preference and a technical cache (Lesson 04) are stored — the banner exists for transparency/teaching purposes, not because there's tracking to hide.

Activated on every page with:

<script src="js/cookieConsent.js"></script>
<script>initCookieConsent();</script>

Source: [Source: ../js/cookieConsent.js].
