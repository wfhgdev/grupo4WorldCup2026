💾 Sistema de Caché

📖 Para principiantes: conceptos básicos

1. localStorage — Almacenamiento persistente en el navegador. Los datos sobreviven al reinicio.
Límite: ~5MB por dominio.
2. TTL — Time To Live. Tiempo de vida de la caché. En el proyecto son 5 minutos.
3. JSON.parse/stringify — Conversión entre objeto y cadena.
localStorage solo guarda texto. Los objetos hay que convertirlos.
4. Estrategia de caché — Primero revisar localStorage. Si los datos existen y no están obsoletos, úsalos. Si no, solicita a la API, guarda en localStorage, devuelve.

¿Qué es la caché?

Es memoria temporal para no repetir la misma consulta una y otra vez. En este proyecto guardamos respuestas JSON en localStorage con TTL de 5 minutos. Código real: [Fuente: ../js/api.js].

💾 Cache System

📖 For beginners: key concepts

1. localStorage — Persistent browser storage. Data survives restarts. Limit: ~5MB per domain.
2. TTL — Time To Live. How long the cache is valid. In this project: 5 minutes.
3. JSON.parse/stringify — Convert between objects and strings. localStorage only stores strings, so objects must be converted.
4. Cache strategy — Check localStorage first. If data exists and is fresh, use it. If not, fetch from API, save to localStorage, return.

What is Cache?

Temporary memory to avoid repeating the same request over and over. In this project we store JSON responses in localStorage with a 5-minute TTL. Real code: [Source: ../js/api.js].
