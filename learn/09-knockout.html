🏆 Página de Eliminatorias

📖 Para principiantes:
  • Promise.all — lanzar varias peticiones en paralelo. Espera todas, devuelve un array de resultados.
  • Bracket (cuadro del torneo) — árbol de partidos: 1/8 → 1/4 → 1/2 → final.
  • opacity — transparencia. El perdedor recibe opacity-60 (60% de visibilidad).

¿Qué hace esta página?

Representa el árbol de torneo desde octavos hasta la final. Fuentes: [Fuente: ../knockout/knockout.html] y [Fuente: ../knockout/knockout.js].

Construcción del bracket

  1. Obtener partidos por ronda desde la API.
  2. Mapearlos a posiciones fijas del bracket.
  3. Rellenar tarjetas con fillMatchCard().

Para cargar datos en paralelo puedes usar Promise.all.

const [a, b, c] = await Promise.all([
  fetch('/api/left'),
  fetch('/api/right'),
  fetch('/api/final')
]);

[Fuente: ../knockout/knockout.js] · [Promise.all — MDN ES] · [Promise.all — MDN EN]

⚠️ Importante: si una llamada falla, Promise.all falla completo. Si necesitas resiliencia parcial, usa Promise.allSettled().

Winner highlighting

Se atenúa al perdedor para mantener foco visual en el ganador:

if (homeScore > awayScore) {
  awayDiv.classList.add('opacity-60');
} else if (awayScore > homeScore) {
  homeDiv.classList.add('opacity-60');
}

🏆 Knockout Page

📖 For beginners:
  • Promise.all — launch multiple requests in parallel. Waits for all, returns an array of results.
  • Bracket (tournament grid) — match tree: 1/8 → 1/4 → 1/2 → final.
  • opacity — transparency. The loser gets opacity-60 (60% visibility).

What it does

Renders a tournament bracket from round of 16 to final. Source: [Source: ../knockout/knockout.html], [Source: ../knockout/knockout.js].

Bracket mapping

Map fetched matches to fixed bracket positions and fill cards with fillMatchCard().
