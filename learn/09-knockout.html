🏆 Página de Eliminatorias

📖 Para principiantes:
  • Promise.all — lanzar varias peticiones en paralelo. Espera todas, devuelve un array de resultados.
  • Bracket (cuadro del torneo) — árbol de partidos: 1/8 → 1/4 → 1/2 → final.
  • opacity — transparencia. El perdedor recibe opacity-60 (60% de visibilidad).

¿Qué hace esta página?

Representa el árbol de torneo completo, desde dieciseisavos (LAST_32) hasta la final. Fuentes: [Fuente: ../knockout/knockout.html] y [Fuente: ../knockout/knockout.js]. Como en el resto de páginas, comparte el mismo footer inyectado por fetch (ver Lección 12).

Construcción del bracket

  1. Pedir las seis rondas del torneo en paralelo con Promise.all.
  2. Ordenar cada ronda por fecha y repartirla entre lado izquierdo/derecho del bracket.
  3. Mapear cada partido a una posición fija (data-match-id) del bracket.
  4. Rellenar cada tarjeta con fillMatchCard().

El código real usa getMatchesByStage() de js/api.js, no rutas inventadas:

const [last32Data, last16Data, quarterData, semiData, thirdData, finalData] = await Promise.all([
  getMatchesByStage('LAST_32'),
  getMatchesByStage('LAST_16'),
  getMatchesByStage('QUARTER_FINALS'),
  getMatchesByStage('SEMI_FINALS'),
  getMatchesByStage('THIRD_PLACE'),
  getMatchesByStage('FINAL')
]);

[Fuente: ../knockout/knockout.js] · [Promise.all — MDN ES] · [Promise.all — MDN EN]

⚠️ Importante: si una llamada falla, Promise.all falla completo. Si necesitas resiliencia parcial, usa Promise.allSettled().

Conectores SVG del bracket

Las líneas que unen partidos no son imágenes: son un <svg id="bracket-connectors"> dibujado por JavaScript. drawBracketConnectors() lee la posición real (getBoundingClientRect()) de cada pareja de tarjetas conectadas y traza un <path> en forma de escuadra entre ellas.

const BRACKET_CONNECTIONS = [
  [15, 0], [16, 0], /* ... */ [4, 13], [5, 13], [13, 8]
];

path.setAttribute('d', `M ${startX} ${startY} H ${bendX} V ${endY} H ${endX}`);

Como el layout cambia con el tamaño de pantalla, un ResizeObserver sobre el contenedor del bracket vuelve a dibujar los conectores cada vez que cambian las dimensiones (además de escuchar resize de la ventana):

bracketConnectorObserver = new ResizeObserver(scheduleBracketConnectorDraw);
bracketConnectorObserver.observe(canvas);
window.addEventListener('resize', scheduleBracketConnectorDraw);

🏆 Knockout Page

📖 For beginners:
  • Promise.all — launch multiple requests in parallel. Waits for all, returns an array of results.
  • Bracket (tournament grid) — match tree: 1/8 → 1/4 → 1/2 → final.
  • opacity — transparency. The loser gets opacity-60 (60% visibility).

What it does

Renders the full tournament bracket, from round of 32 (LAST_32) to the final. Source: [Source: ../knockout/knockout.html], [Source: ../knockout/knockout.js]. Like other pages, it shares the same fetch-injected footer (see Lesson 12).

Bracket mapping

Fetch six stages in parallel with real helpers, map fetched matches to fixed bracket positions and fill cards with fillMatchCard():

const [last32Data, last16Data, quarterData, semiData, thirdData, finalData] = await Promise.all([
  getMatchesByStage('LAST_32'),
  getMatchesByStage('LAST_16'),
  getMatchesByStage('QUARTER_FINALS'),
  getMatchesByStage('SEMI_FINALS'),
  getMatchesByStage('THIRD_PLACE'),
  getMatchesByStage('FINAL')
]);

SVG bracket connectors

The connecting lines are a JS-drawn <svg id="bracket-connectors">, not images. drawBracketConnectors() reads each pair's getBoundingClientRect() and traces an elbow <path> between them. A ResizeObserver on the bracket container redraws connectors whenever the layout changes (plus a window resize listener).
