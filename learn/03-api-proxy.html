🔌 API & Proxy (JS / Serverless)

📖 Para principiantes: conceptos básicos

1. HTTP — Protocolo de transferencia de datos en internet. Cuando el navegador pide una página, envía una solicitud HTTP. El servidor responde con una respuesta HTTP.
Métodos: GET (obtener), POST (enviar), PUT (actualizar), DELETE (eliminar).
Códigos de respuesta: 200 OK, 404 Not Found, 500 Server Error.
2. API Endpoint — Una URL concreta desde la que se pueden obtener datos.
https://api.football-data.org/v4/competitions/WC/matches
https://api.football-data.org/v4/competitions/WC/scorers
3. JSON — Formato de datos. Es similar a un objeto de JavaScript, pero es texto.
{ "matches": [ { "id": 1, "status": "FINISHED" } ] }
4. Token (API key) — Clave secreta para acceder a la API. football-data.org requiere la cabecera X-Auth-Token.
5. CORS — El navegador bloquea peticiones a otro dominio si éste no lo permite. El proxy evita esto porque la petición va desde el servidor, no desde el navegador.

¿Qué problema resuelve el proxy?

El navegador bloquea peticiones a dominios distintos por seguridad (CORS). Nuestro proxy recibe la petición, la reenvía a la API y devuelve la respuesta. Así el navegador solo ve tráfico a tu propio servidor. Código real: [Fuente: ../api/proxy.js].

Recurso: MDN — CORS en español · MDN — CORS EN.

Flujo de una llamada

1. fetch('/api/proxy?endpoint=competitions/WC/matches')
2. La función serverless (api/proxy.js) recibe la petición
3. Añade cabeceras (X-Auth-Token, User-Agent)
4. Reenvía con fetch() a api.football-data.org/v4/...
5. Devuelve la respuesta JSON al navegador

Las funciones de ayuda están en [Fuente: ../js/api.js].

Recurso: MDN — Fetch ES · MDN — Fetch EN.

Seguridad: ocultar tokens

El token se guarda en el servidor (en la función serverless api/proxy.js), no en el HTML. Si alguien ve tu HTML solo verá la llamada al proxy, no la clave secreta.

Recurso: MDN — Seguridad web.

⚠️ Importante: nunca expongas tokens en el frontend. Úsalos solo en el servidor o en variables de entorno.

🔌 API & Proxy (JS / Serverless)

📖 For beginners: basic concepts

1. HTTP — Data transfer protocol on the internet. When the browser requests a page, it sends an HTTP request. The server replies with an HTTP response.
Methods: GET (obtain), POST (send), PUT (update), DELETE (delete).
Response codes: 200 OK, 404 Not Found, 500 Server Error.
2. API Endpoint — A specific URL from which you can get data.
https://api.football-data.org/v4/competitions/WC/matches
https://api.football-data.org/v4/competitions/WC/scorers
3. JSON — Data format. It looks like a JavaScript object, but it's text.
{ "matches": [ { "id": 1, "status": "FINISHED" } ] }
4. Token (API key) — Secret key to access the API. football-data.org requires the X-Auth-Token header.
5. CORS — The browser blocks requests to another domain if it doesn't allow it. The proxy bypasses this because the request goes from the server, not the browser.

What problem does the proxy solve?

The browser blocks requests to different domains for security (CORS). Our proxy receives the request, forwards it to the API, and returns the response. The browser only sees traffic to your own server. Real code: [Source: ../api/proxy.js].

Resource: MDN — CORS EN.

Request flow

1. fetch('/api/proxy?endpoint=competitions/WC/matches')
2. The serverless function (api/proxy.js) receives the request
3. Adds headers (X-Auth-Token, User-Agent)
4. Forwards via fetch() to api.football-data.org/v4/...
5. Returns JSON response to the browser

Helper functions in [Source: ../js/api.js].

Resource: MDN — Fetch EN.

Security: hiding tokens

The token is stored on the server (in the serverless function api/proxy.js), not in the HTML. If someone inspects your HTML, they only see the proxy call, not the secret key.

Resource: MDN — Web security.

⚠️ Important: never expose tokens in the frontend. Use them only on the server or in environment variables.
