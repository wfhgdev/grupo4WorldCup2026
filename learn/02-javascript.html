⚡ JavaScript

📖 Para principiantes: conceptos básicos de JS
  1. Variable — Una caja con nombre donde guardas un valor.
    let nombre = 'Brasil'; // string
    const PI = 3.14; // number, no se puede reasignar
    var viejo = 'no usar'; // obsoleto
    let equipos = []; // array vacío
    let datos = {}; // objeto vacío
  2. Array — Lista ordenada de valores. El índice empieza en 0.
    let paises = ['Brasil', 'Argentina', 'España'];
    paises[0] → 'Brasil'  |  paises.length → 3  |  paises.push('México') // añade al final
  3. Objeto — Colección de propiedades clave: valor.
    let equipo = { nombre: 'Brasil', goles: 2, clasificado: true };
    equipo.nombre → 'Brasil'  |  equipo['goles'] → 2
  4. Función — Bloque de código reutilizable que se invoca por su nombre.
    function sumar(a, b) { return a + b; }
    sumar(2, 3) → 5
    const restar = (a, b) => a - b; // arrow function
  5. DOM — Document Object Model. El navegador convierte el HTML en un árbol de objetos con los que JS puede interactuar:
    document.getElementById('id')
    document.querySelector('.clase')
    document.querySelectorAll('div')
  6. Evento — Acción del usuario: clic, tecla pulsada, envío de formulario.
    button.addEventListener('click', () => { alert('¡Click!'); });
  7. Promise — "Todavía no sé el resultado, pero lo sabré pronto". Tiene 3 estados: pending (esperando), resolved (éxito), rejected (error).
  8. async/await — Azúcar sintáctico para Promises. async marca una función como asíncrona; await espera el resultado.
    async function cargarDatos() {
      const respuesta = await fetch('url');
      const datos = await respuesta.json();
      return datos;
    }

¿Qué es JavaScript?

JavaScript es el lenguaje que convierte una página estática en una experiencia interactiva: responde clics, carga datos y actualiza contenido sin recargar. En este proyecto usamos JavaScript "Vanilla", sin frameworks adicionales.

Fuente del proyecto en [Fuente: ../js/api.js].

Recurso: MDN — JavaScript en español.

DOM: relación entre HTML y JS

El DOM representa el HTML como un árbol de nodos. Con JS puedes:

  • Buscar elementos por id, clase o selector MDN — getElementById
  • Crear, modificar y eliminar nodos MDN — createElement
  • Responder a eventos del usuario MDN — addEventListener
// Buscar por id
const menu = document.getElementById('mobile-menu-overlay');

// Buscar por clase
const cards = document.querySelectorAll('.match-card');

// Crear elemento
const div = document.createElement('div');

// Modificar clases
div.classList.add('hidden');
div.classList.remove('hidden');

// Escuchar un clic
button.addEventListener('click', () => {
  menu.classList.remove('hidden');
});
💡 Regla: usa querySelector('#id') o querySelector('.clase') para un solo elemento y querySelectorAll para muchos.

Async / Await

Sirve para esperar sin congelar la página. async declara la función; await pausa esa línea hasta resolver la operación.

async function obtenerDatos() {
  try {
    const respuesta = await fetch('https://api.ejemplo.com/datos');
    const datos = await respuesta.json();
    return datos;
  } catch (error) {
    console.error('Error:', error);
  }
}

En el proyecto se usa para cargar partidos, grupos, eliminatorias y estadísticas desde [Fuente: ../js/api.js].

Recurso: MDN — async function.

Fetch API

fetch() hace peticiones HTTP desde el navegador. En este proyecto todas las llamadas a datos deportivos pasan por el proxy serverless (football-data.org no permite CORS desde el navegador):

// fetchApi() en js/api.js hace esto por ti:
const response = await fetch('/api/proxy?endpoint=' + encodeURIComponent('/competitions/WC/matches'));
const data = await response.json();

// En la práctica usas las funciones ya envueltas:
const data = await getAllMatches(); // ver Lección 03
⚠️ Error común: fetch() solo lanza error en fallos de red; para errores HTTP debes revisar response.ok. En este proyecto fetchApi() ya hace esa comprobación por ti.

Promise.all

Ejecuta varias llamadas en paralelo y espera todas. La página de eliminatorias la usa para pedir seis rondas del torneo a la vez:

const [last32, last16, quarters, semis, third, final] = await Promise.all([
  getMatchesByStage('LAST_32'),
  getMatchesByStage('LAST_16'),
  getMatchesByStage('QUARTER_FINALS'),
  getMatchesByStage('SEMI_FINALS'),
  getMatchesByStage('THIRD_PLACE'),
  getMatchesByStage('FINAL')
]);

⚡ JavaScript

📖 For beginners: basic JS concepts
  1. Variable — A named box where you store a value.
    let name = 'Brazil'; // string
    const PI = 3.14; // number, cannot be reassigned
    var old = 'do not use'; // deprecated
    let teams = []; // empty array
    let data = {}; // empty object
  2. Array — Ordered list of values. Index starts at 0.
    let countries = ['Brazil', 'Argentina', 'Spain'];
    countries[0] → 'Brazil'  |  countries.length → 3  |  countries.push('Mexico') // adds to end
  3. Object — Collection of key: value properties.
    let team = { name: 'Brazil', goals: 2, qualified: true };
    team.name → 'Brazil'  |  team['goals'] → 2
  4. Function — Reusable block of code called by its name.
    function add(a, b) { return a + b; }
    add(2, 3) → 5
    const subtract = (a, b) => a - b; // arrow function
  5. DOM — Document Object Model. The browser turns HTML into an object tree JS can interact with:
    document.getElementById('id')
    document.querySelector('.class')
    document.querySelectorAll('div')
  6. Event — User action: click, key press, form submission.
    button.addEventListener('click', () => { alert('Clicked!'); });
  7. Promise — "I don't know the result yet, but I will soon". 3 states: pending (waiting), resolved (success), rejected (error).
  8. async/await — Syntactic sugar for Promises. async marks a function as async; await waits for the result.
    async function loadData() {
      const response = await fetch('url');
      const data = await response.json();
      return data;
    }

The DOM represents HTML as a tree of nodes. With JS you can:

  • Find elements by id, class, or selector MDN — getElementById

Async / Await

Used to wait without freezing the page. async declares the function; await pauses that line until the operation resolves.

async function fetchData() {
  try {
    const response = await fetch('https://api.example.com/data');
    const data = await response.json();
    return data;
  } catch (error) {
    console.error('Error:', error);
  }
}

In the project it's used to load matches, groups, knockout and stats from [Source: ../js/api.js].

Resource: MDN — Asynchronous JavaScript.

Fetch API

fetch() makes HTTP requests from the browser. In this project every sports-data call goes through the serverless proxy (football-data.org doesn't allow CORS from the browser):

Project source: [Source: ../js/api.js].

// fetchApi() in js/api.js does this for you:
const response = await fetch('/api/proxy?endpoint=' + encodeURIComponent('/competitions/WC/matches'));
const data = await response.json();

// In practice you call the already-wrapped helpers:
const data = await getAllMatches(); // see Lesson 03
⚠️ Common mistake: fetch() only throws on network failures; for HTTP errors you must check response.ok. In this project fetchApi() already does that check for you.
